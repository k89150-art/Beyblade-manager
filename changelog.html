v1.8.0｜全站介面更新與卡片優化
2026-07-15～2026-07-16
  • 介面重新設計為手機優先的收藏與配置管理介面，使用較大的操作區、清楚的資訊層級與一致的科技競技風格。
  • 導覽桌面版改用固定左側功能欄，手機版新增收藏、庫存、配置、分析與賽事底部導覽。
  • 手機收藏、額外零件、配置與歷史紀錄在手機上改為卡片顯示，不再需要橫向拖曳才能查看全部欄位。
  • 精簡收藏與配置卡片只顯示實際有輸入的欄位，BX、UX 與 CX 都不再以空白欄位撐高卡片。
  • 桌面收藏、配置與歷史紀錄改用卡片網格呈現，保留原有表格資料與編輯流程。
  • 配置桌面配置卡改為試作稿的緊湊資訊卡，集中顯示系列、型號、組合摘要與操作按鈕。
  • 卡片配置卡重新套用確認過的試作層級：系列徽章、型號名稱、完整配置摘要、結構標籤與底部操作列。
  • 對齊統一配置卡標題、摘要與結構標籤的左側基準線，並移除沒有功能的省略號圖示。
  • 排列桌面配置卡改為自動欄數，依可用寬度顯示 2、3 或 4 張，減少右側空白。
  • CX分體式 CX 將金屬戰刃與超越戰刃合併為同一資訊組，零件值會與「紋章鎖、金屬+超越、輔助、固鎖、軸心」標籤垂直對齊。
  • 名稱CX 收藏與配置顯示改為依實際分件判斷；分體式名稱優先使用金屬戰刃，沒有 CX 分件的商品則沿用上蓋、固鎖與軸心結構。
  • 相容舊版 CX 整合上蓋資料會從配置列或相同型號收藏資料補回名稱與標籤，不再只顯示固鎖、軸心。
  • 標籤BX／UX 與舊式 CX 卡片統一顯示上蓋、固鎖、軸心；分體式 CX 則顯示紋章鎖、金屬+超越、輔助、固鎖與軸心。
  • 新增建立配置改為按需開啟的快速編輯面板，桌面由右側滑入、手機使用全螢幕表單，不再長期占用配置頁上方空間。
  • 導覽電腦版左側欄可收合成圖示列，並會記住本機瀏覽器的收合狀態。
  • 歷史桌面歷史測試改為橫向紀錄卡，日期、結果、備註與還原操作更容易掃讀。
  • 分析展開單筆庫存建議時，不再連帶拉高同列的其他建議卡片。
  • 賽事手機版回合紀錄改為緊湊卡片，修正表格寬度造成的內容偏移與橫向裁切。
  • 庫存額外零件庫存區只顯示使用者自行新增的零件，並新增種類、持有、使用中與可用數量摘要。
  • 保留Google 登入、Firestore 資料、CX 規則、數量防呆、配置修改刪除與歷史還原流程全部沿用原有邏輯。
  • 備份更新前版本已保存於 backup/pre-ui-redesign-20260715 分支，可隨時回復。
v1.7.0｜2026-07-11 資料庫與分析規則更新
2026-07-11
  • 資料合併 2026-07-11 Beyblade X 增量資料，新增 BX-49 蒼龍突擊、CX-18 腕龍鞭擊與多筆主流、次主流、特殊成功配置路線。
  • 規則新增 7 筆 CX priority rules，命中優先規則時會鎖定定位，避免被一般分數誤判。
  • 名稱修正名稱與縮寫對照：I 為 Ignition、Y 為 Yielding、極變為 Delta；Bit 分析句維持使用縮寫。
  • 分析調整分析流程為 alias 正規化、基礎向量、priority rules、情境規則、衝突偵測、同尺度計分與語意去重。
  • 推薦successful_rogue 與 single_sample 路線只作測試提示，不列為主要推薦，也不自動判定為路線衝突。
  • 資料量分析資料庫更新後：Blade 77 筆、Part 12 筆、Priority Rule 7 筆、Alias 8 筆、Bit 50 筆。
  • 測試完成 2026-07-11 regression tests：名稱顯示 5/5、Bit 顯示 4/4、分析邏輯 9/9 全部通過。
  • 暫緩依目前資料策略，BX-50 與 UX-20 暫時不加入資料庫。
v1.6.0｜配置分析資料 v1.8 與推薦體驗優化
2026-07-01
  • 資料整合 Beyblade X Codex v1.8 資料包，優先使用 ASCII-safe 資料庫，降低中文亂碼風險。
  • 分析一般 BX / UX 配置分析優先使用 v1.8 helper；CX 分析保留既有穩定相容邏輯作為 fallback。
  • 資料量分析資料庫擴充至上蓋 50 筆、固鎖 37 筆、軸心 49 筆，並補入 CX 主要戰刃、金屬戰刃、Over 與輔助戰刃資料。
  • 快捷第三區配置紀錄每列新增「分析」按鈕，可直接帶入配置分析頁並自動分析，不會修改資料或扣除庫存。
  • 手機庫存推薦結果改為可展開卡片，手機版預設只展開第一筆，減少畫面過長。
  • 成本庫存推薦加入 60 秒頁面快取，短時間重複點擊不會重新讀取 Firestore。
  • 提醒補強說明：配置分析與推薦僅供理論參考，不保證實戰一定有效或取得勝利。
  • 備份本次更新後可直接從 GitHub 下載完整專案作為備份檔。
v1.5.7｜配置分析
2026-06-30
  • 新增新增配置分析頁，可輸入 BX / UX 配置或 CX 三件式、CX 分體式配置進行分析。
  • CXCX 分析會區分「紋章鎖 + 主要戰刃 + 輔助戰刃」與「紋章鎖 + 金屬戰刃 + 超越戰刃 + 輔助戰刃」。
  • 防呆Op / Tr 固鎖一體式軸心會提示不能再搭配一般固鎖。
  • 引擎接入 analyzeCombo() 規則引擎，顯示一句話定位、七維分數、改裝建議、3G / 5G 建議位置與信心等級。
  • 說明分析結果只做理論判讀，不會保存、不會修改配置紀錄，也不會扣除庫存。

  • 防呆選擇 Tr 軸時，若同時選擇固鎖將無法保存。

v1.5.5｜結構整理與手機操作優化
2026-06-28
  • 結構將 UX-19 與 Tr 軸無固鎖配置規則整合回主程式，移除外掛補丁引用。
  • 結構停用舊版 prototype 補丁，降低之後維護與除錯風險。
  • 手機編輯資料列時新增高亮提示，手機操作更容易辨識目前正在修改哪一筆。
  • 手機表格操作欄不固定在右側，修改與刪除按鈕改為同列緊湊排列，保留更多橫向可視欄位空間。
  • 部署新增 Firestore Rules 手動部署流程，降低規則更新後忘記發布的機率。
v1.5.4｜操作體驗與資料顯示優化
  • 改善資料載入提示，雲端資料同步時會顯示載入狀態，減少誤以為資料消失的情況。
  • 新增管理員後台，可由管理員唯讀檢視使用者資料摘要與個別資料內容。
  • 工具頁管理員入口改為只有管理員登入後顯示，一般使用者不會看到管理員後台連結。
  • 修正 CX 分體戰刃配置在修改時可能觸發二選一限制的問題。
  • 管理員後台新增 Email 欄位，方便辨識不同使用者資料。
  • 補強 Firestore 規則，使用者僅能讀寫自己的 main 資料，管理員僅能讀取 main 資料。
v1.5.3｜配置排序與表格優化
  • 配置紀錄區排序改為與上方陀螺配置相同規則：UX → BX → CX → 其他，且同系列依型號數字排序。
  • 配置紀錄排序功能整合進主程式，移除額外排序檔案載入，降低功能重複與衝突機率。
  • 優化工具頁表格樣式與操作按鈕間距，讓陀螺配置、零件庫存、配置紀錄與歷史紀錄表格更一致。
  • 補強特殊配置支援：Tr 軸與 UX-19 子彈獅鷲可使用「無固鎖（-）」配置。
v1.5.2｜配置庫存檢查修正
  • 修正抽包系列新增配置時，未選擇主要戰刃仍可能被判定為庫存不足的問題。
  • 選填零件未選擇時，系統會正確視為空值，不再將「-」當成實際零件檢查。
